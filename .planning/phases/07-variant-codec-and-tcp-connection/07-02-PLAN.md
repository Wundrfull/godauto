<objective>
Build the protocol message framing layer and async TCP session with background receive loop.

Purpose: The protocol layer wraps Variant-encoded data in the correct wire format (length-prefixed 3-element Arrays) that Godot's debugger expects. The session manages the TCP server lifecycle, accepts a game connection, and continuously drains messages (including unsolicited performance data at ~60/sec) to prevent buffer flooding that would freeze the game.

Output: `src/gdauto/debugger/protocol.py` (message framing), `src/gdauto/debugger/session.py` (async TCP session), unit tests for both.
</objective>

  <action>
    Create `src/gdauto/debugger/protocol.py` with three functions:

    1. `encode_message(command: str, data: list, thread_id: int = 1) -> bytes`:
       - Build the 3-element array: `[command, thread_id, data]`
       - Variant-encode the array: `payload = variant.encode([command, thread_id, data])`
       - Prepend 4-byte LE uint32 length: `struct.pack('<I', len(payload)) + payload`
       - Return the complete framed message

    2. `decode_message(payload: bytes) -> tuple[str, int, list]`:
       - Variant-decode the payload (no length prefix; that's stripped by read_message)
       - Extract the 3-element array
       - Validate: array must have exactly 3 elements, element 0 must be str, element 1 must be int, element 2 must be list
       - Raise ProtocolError if validation fails with code "PROTO_INVALID_MESSAGE" and fix "Expected 3-element array [command: str, thread_id: int, data: list]"
       - Return (command_name, thread_id, data_array)

    3. `async def read_message(reader: asyncio.StreamReader) -> tuple[str, int, list]`:
       - Read 4 bytes for length: `size_data = await reader.readexactly(4)`
       - Unpack: `size = struct.unpack('<I', size_data)[0]`
       - Validate size <= 8_388_608 (8 MiB); raise ProtocolError with code "PROTO_MSG_TOO_LARGE" and fix "This usually indicates a protocol synchronization error" if exceeded
       - Read payload: `payload = await reader.readexactly(size)`
       - Return `decode_message(payload)`

    Also add a convenience:
    4. `async def write_message(writer: asyncio.StreamWriter, command: str, data: list, thread_id: int = 1) -> None`:
       - `framed = encode_message(command, data, thread_id)`
       - `writer.write(framed)`
       - `await writer.drain()`

    Imports: `from __future__ import annotations`, `import asyncio`, `import struct`, `from gdauto.debugger.variant import encode, decode`, `from gdauto.debugger.errors import ProtocolError`

    Create `tests/unit/test_protocol.py` with tests:
    - `test_encode_message_structure`: encode_message("test", []) produces bytes starting with 4-byte length prefix
    - `test_encode_message_contains_command`: decode the payload after the length prefix; the decoded array's first element is "test"
    - `test_encode_message_contains_thread_id`: default thread_id is 1; custom thread_id 42 appears in decoded array
    - `test_encode_message_contains_data`: data [1, "hello"] appears as third element in decoded array
    - `test_decode_message_round_trip`: decode_message(encode_message("cmd", ["arg1", 42])[4:]) returns ("cmd", 1, ["arg1", 42])
    - `test_decode_message_invalid_element_count`: decode_message with 2-element array raises ProtocolError
    - `test_decode_message_invalid_types`: decode_message with non-string command raises ProtocolError
    - `test_read_message_size_limit`: create an asyncio.StreamReader, feed it a length prefix of 9_000_000, verify ProtocolError raised
    - `test_read_message_happy_path`: feed valid encoded message into StreamReader, verify read_message returns correct tuple
    - `test_write_message`: capture bytes written to a mock writer, verify they match encode_message output

    Create `src/gdauto/debugger/session.py` with class `DebugSession`:

    ```python
    @dataclass
    class DebugSession:
        port: int = 6007
        host: str = "127.0.0.1"
    ```

    Instance attributes (set in __init__ or __post_init__):
    - `_server: asyncio.Server | None = None`
    - `_reader: asyncio.StreamReader | None = None`
    - `_writer: asyncio.StreamWriter | None = None`
    - `_connected: asyncio.Event` (set when a game connects)
    - `_recv_task: asyncio.Task | None = None`
    - `_thread_id: int | None = None` (captured from first received message)
    - `_pending: dict[str, asyncio.Future]` (command -> response future)
    - `_output_buffer: list[list]` (capped at 1000 entries)
    - `_error_buffer: list[list]` (capped at 1000 entries)
    - `_closed: bool = False`

    Methods:

    1. `async def start(self) -> None`:
       - `self._server = await asyncio.start_server(self._handle_connection, self.host, self.port)`
       - Set SO_REUSEADDR on the socket for quick reuse after crash (access via `self._server.sockets[0].setsockopt(socket.SOL_SOCKET, socket.SO_REUSEADDR, 1)` -- note: asyncio may set this by default on some platforms, but be explicit)
       - Does NOT wait for connection; returns immediately after server starts listening

    2. `async def wait_for_connection(self, timeout: float = 30.0) -> None`:
       - `await asyncio.wait_for(self._connected.wait(), timeout=timeout)`
       - Raises DebuggerTimeoutError with code "DEBUG_CONNECT_TIMEOUT" and fix "Check that the game launched successfully and can reach {host}:{port}" on timeout

    3. `async def _handle_connection(self, reader: asyncio.StreamReader, writer: asyncio.StreamWriter) -> None`:
       - Store reader/writer
       - `self._connected.set()`
       - `self._recv_task = asyncio.create_task(self._recv_loop())`

    4. `async def _recv_loop(self) -> None`:
       - Loop while not self._closed and reader is not at EOF:
         - `command, thread_id, data = await read_message(self._reader)`
         - Capture thread_id on first message: `if self._thread_id is None: self._thread_id = thread_id`
         - Dispatch:
           - If command in self._pending: `self._pending.pop(command).set_result(data)`
           - elif command == "output": append data to _output_buffer (cap at 1000; drop oldest if full)
           - elif command == "error": append data to _error_buffer (cap at 1000; drop oldest if full)
           - elif command starts with "performance:": discard silently (these arrive ~60/sec)
           - else: discard (unknown unsolicited message)
       - Catch `asyncio.IncompleteReadError`: connection closed by game; set self._closed = True
       - Catch `ProtocolError`: log or buffer the error, continue if connection still open

    5. `async def send_command(self, command: str, data: list | None = None, timeout: float = 10.0) -> list`:
       - data defaults to []
       - thread_id defaults to self._thread_id or 1
       - Create asyncio.Future, store in self._pending[command]
       - `await write_message(self._writer, command, data or [], thread_id)`
       - `result = await asyncio.wait_for(future, timeout=timeout)`
       - Remove from pending on timeout, raise DebuggerTimeoutError with code "DEBUG_CMD_TIMEOUT" and fix f"Command '{command}' did not receive a response within {timeout}s"
       - Return result

    6. `async def close(self) -> None`:
       - self._closed = True
       - Cancel recv_task if running
       - Close writer if connected
       - Close server
       - Reset all state

    7. `@property def connected(self) -> bool`: return self._connected.is_set()
    8. `@property def thread_id(self) -> int | None`: return self._thread_id
    9. `@property def output_buffer(self) -> list[list]`: return list(self._output_buffer)
    10. `@property def error_buffer(self) -> list[list]`: return list(self._error_buffer)

    Async context manager support:
    - `async def __aenter__(self) -> DebugSession`: call self.start(), return self
    - `async def __aexit__(self, *exc) -> None`: call self.close()
