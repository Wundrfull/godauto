<objective>
Build the Variant binary codec that encodes and decodes Godot 4.x types for debugger communication, along with the debugger error hierarchy and data models.

Purpose: The Variant codec is the foundation of all debugger communication. Every message sent to or received from a Godot game is a Variant-encoded binary payload. If the codec is wrong by even one byte, Godot silently drops the message with no diagnostic. This plan uses TDD to build the codec with golden-byte tests that verify byte-exact fidelity against documented Godot encoding rules.

Output: `src/gdauto/debugger/variant.py` (encode/decode), `src/gdauto/debugger/errors.py` (error hierarchy), `src/gdauto/debugger/models.py` (type wrappers), exhaustive test suite.
</objective>

Order: None -> nil, bool -> bool (before int since bool is int subclass), GodotStringName -> string_name, GodotNodePath -> node_path, int -> int, float -> float, str -> string, list -> array, dict -> dictionary, bytes -> packed_byte_array, tuple with type_hint -> vector/color/etc.

       - For OBJECT: if flag indicates as_id, read uint64; else read uint32 (0 = null)
       - For Packed*Arrays: read count, then read count elements

    7. Public `decode(data, offset=0) -> tuple[object, int]`: read header, extract base_type, dispatch. Raise ProtocolError for unknown types and truncated data.
