<objective>
Build the core Aseprite-to-SpriteFrames data pipeline: parse Aseprite JSON exports into typed dataclasses and transform them into Godot SpriteFrames GdResource instances ready for .tres serialization.

Purpose: This is the foundation for the entire phase. The Aseprite parser provides the input data model, and the SpriteFrames builder provides the transformation logic. Both are pure Python with zero external dependencies and are consumed by all subsequent sprite commands.

Output: Two new modules (`formats/aseprite.py`, `sprite/spriteframes.py`), six Aseprite JSON test fixtures, and comprehensive unit tests for both parser and builder.
