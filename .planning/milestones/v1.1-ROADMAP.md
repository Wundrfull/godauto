# Milestone v1.1: Godot 4.6 Compatibility and Audit

**Status:** SHIPPED 2026-03-29
**Phases:** 5-6
**Total Plans:** 4

## Overview

Audit existing v1.0 functionality against Godot 4.6.1, ensure compatibility with engine changes, and establish a 4.5+ baseline with forward support for 4.6.x features.

## Phases

### Phase 5: Format Compatibility and Backwards Safety

**Goal**: Generated .tres/.tscn files match Godot 4.6.1 conventions with no regressions for Godot 4.5 users
**Depends on**: Phase 4 (v1.0 complete)
**Plans**: 2 plans

Plans:
- [x] 05-01: Parser and format layer changes (unique_id, PackedVector4Array, load_steps removal)
- [x] 05-02: Test infrastructure update and golden file regeneration

**Details:**
- Removed load_steps from all 7 generator call sites unconditionally
- Added unique_id field to SceneNode with parse/serialize support
- Added PackedVector4Array dataclass with parse/serialize support
- Normalization patterns for load_steps and unique_id stripping in golden file tests
- 17 new format compatibility tests across 4 test classes
- Golden files regenerated without load_steps; 668 unit tests pass

## Milestone Summary

**Key Decisions:**
- Set load_steps=None unconditionally in all generators; Godot 4.5 tolerates omission
- unique_id stored as int; PackedVector4Array uses flat-float pattern
- Normalization uses leading-space-anchored regex for load_steps and unique_id stripping
- Ecosystem table uses category descriptions, not product names, to avoid staleness
- CLI root help text is single source of truth for SKILL.md compatibility claim

**Issues Resolved:**
- Godot 4.6.1 format divergence (load_steps no longer emitted by engine)
- unique_id attribute preservation on scene round-trip
- PackedVector4Array parsing for format=4 files

**Issues Deferred:**
- format=4 .tscn test coverage (parse_tscn with format=4 header; implementation correct, test gap only)
- Godot 4.5 binary-specific E2E test (design limitation; tests run against any >= 4.5)

**Technical Debt Incurred:**
- ROADMAP.md progress table had stale Phase 5 status (cosmetic)
